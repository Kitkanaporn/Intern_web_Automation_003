# Promote_Notification Workflow

## files to use

- `AGENT.md` -> overview and instructions of project 
- `SKILL.md` -> about setup in first time and oeration in each phase start at 0 to 6 
- `shared_config.yaml` — global email and schedule config
- `project_config.yaml` — Jira project CSD config
- `Map_User_Email.xlsx` — user-to-email mapping (use the "map" sheet)
- `Promote_Template_Email.html` — email HTML template

## Steps

1. Read `AGENT.md` and `SKILL/md` to load workflow logic.
2. Read `shared_config.yaml` and `project_config.yaml` to get all config values
    2.1. if information is empty that system will ask information from user to hold in blank.
3. Connect to Jira and fetch all Epics issues from the CSD project where :
    - Promote Date is within the next 3 days 
    - Epic that unnotification (notification = "Promote Notification Complete") and has promote date in period (Promote date - 3 <= today < Promote date ) 
4. Cross-reference with `Map_User_Email.xlsx` to find email address of service unit reference by each Epics
5. Execute Pahse 0-6 as defined in `SKILL.md` - fully automated ,no comfirmeation needed.

## Output rules

1. result illustates on screen only - do **Not generate result in file systle** 
2. Run fully sutomatically without asking for confirmation at any step 
    2.1. exceot for the initial step to setup and asking for fill empty information
